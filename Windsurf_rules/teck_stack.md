## Frontend
- **Frameworks**: React 17.x, React Router 5.3.x
- **State Management**: React Query 3.6.x for server state, Redux 4.1.x with Redux Thunk for global state
- **UI Components**: Digit UI Components library
- **Form Handling**: React Hook Form 6.15.x
- **Internationalization**: React i18next 11.16.x
- **Architecture**: Micro-frontend architecture with module-based approach

## Backend
- **Language**: Java 17
- **Framework**: Spring Boot 3.2.x
- **Build Tool**: Maven
- **Architecture**: Microservices with domain-driven design

## Database and Persistence
- **Database**: PostgreSQL
- **ORM**: Spring Data JPA
- **Migrations**: Flyway
- **Caching**: Redis

## API Communication
- **Frontend HTTP Client**: Axios
- **API Design**: RESTful with JSON data format
- **Documentation**: Swagger/OpenAPI
- **Versioning**: Versioned APIs (v1, v2, etc.)

## Messaging and Event Handling
- **Event Streaming**: Kafka
- **Topic Naming Convention**: `<domain>.<action>.<entity>`

## Authentication and Authorization
- **Authentication**: OAuth 2.0
- **Token Format**: JWT
- **Access Control**: Role-based (RBAC)
- **Security**: Encrypted storage of sensitive data

## Testing
- **Backend**: JUnit 5, Mockito
- **Frontend**: Jest, React Testing Library
- **Coverage**: Minimum 80% code coverage required

## Environment Configuration
- **Config Management**: Environment-specific properties files
- **Secrets**: Secure vaults, no hardcoded credentials

## Code Quality and Standards
- **Linting**: ESLint for JavaScript/React
- **Formatting**: Prettier
- **Analysis**: SonarQube
- **Commits**: Conventional Commits format

## CI/CD
- **Pipeline Tool**: Jenkins
- **Process**: Automated testing, code quality gates
- **Versioning**: Semantic versioning for releases

## Documentation
- **Module Documentation**: README.md required for all modules
- **API Documentation**: OpenAPI/Swagger
- **Code Documentation**: Comments for complex logic
- **Architecture**: Architecture Decision Records (ADRs) for significant decisions
